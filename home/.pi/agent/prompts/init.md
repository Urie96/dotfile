请分析此代码库并创建一个 AGENTS.md 文件，该文件将提供给未来的 AI 实例以便在此仓库中操作。

需要添加的内容：

1. 项目特有的常用命令，例如如何构建、代码检查和运行测试。包括在此代码库中进行开发所需的必要命令，例如如何运行单个测试。
2. 高层代码架构和结构，以便未来的实例能够更快地提高生产力。重点关注需要阅读多个文件才能理解的"全局"架构。

使用说明：

- 如果已经存在 AGENTS.md，请提出改进建议。
- 在创建初始 AGENTS.md 时，不要包含显而易见的说明，如"向用户提供有用的错误消息"、"为所有新工具编写单元测试"、"永远不要在代码或提交中包含敏感信息（API 密钥、令牌）"。
- 避免列出可以轻松发现的每个组件或文件结构。
- 不要包含通用的开发实践。
- 不要列出通用的构建/运行命令（如 `cargo build`、`go run`、`npm run build` 等），只有项目特有的非常规命令才需要列出来。
- 如果存在 Cursor 规则（在 .cursor/rules/ 或 .cursorrules 中）或 Copilot 规则（在 .github/copilot-instructions.md 中），请确保包含重要部分。
- 如果存在 README.md，请确保包含重要部分。
- 除非在其他文件中明确包含，否则不要编造诸如"常见开发任务"、"开发技巧"、"支持和文档"之类的信息。
- 请使用中文编写 AGENTS.md，所有标题、说明和描述都应使用中文。技术术语、文件路径、命令和代码示例可保留英文。
